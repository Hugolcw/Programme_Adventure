# PTA题目风格分析

## 📋 风格特点总结

基于对 `guide_of_cumputers` 文件夹下所有PTA题目的分析，总结出以下风格特点：

### 一、输入输出格式特点

#### 1. 输入格式
- **测试用例数量**：
  - 有些题目第一行是测试用例数量 `t`，然后循环 `t` 次
  - 有些题目直接输入数据，处理到文件尾 `while(cin>>n)`
  - 有些题目只有一个测试用例，直接输入

- **输入方式**：
  - 单行输入：`cin >> a >> b`
  - 多行输入：每行一个数据或一组数据
  - 整行输入：`getline(cin, s)` 用于字符串

#### 2. 输出格式
- **非常严格**：精确到空格、换行、标点符号
- **格式要求**：
  - 数字输出：可能要求保留小数位数 `setprecision(2)`
  - 字符串输出：可能有特定格式（如 "数字:5"、"A等级3人"）
  - 图形输出：空格和换行必须精确

### 二、题目描述特点

#### 1. 描述风格
- **简洁明了**：直接说明要做什么
- **格式规范**：通常包含：
  - 题目描述
  - 输入格式（明确说明）
  - 输出格式（明确说明）
  - 样例输入/输出

#### 2. 常见表述
- "输入...，输出..."
- "处理到文件尾"（`while(cin>>n)`）
- "保留X位小数"
- "用空格分隔"

### 三、代码风格特点

#### 1. 代码结构
- **简洁**：通常50-100行
- **直接**：不涉及复杂设计模式
- **实用**：注重算法实现

#### 2. 常用技术

**✅ 可以使用（教学进度内）**：
- **基础数组**：
  - 一维数组：`int a[100]`
  - 二维数组：`int a[100][100]`
  - 固定大小数组
  
- **字符串（基础用法）**：
  - `string`类：基础操作（输入输出、长度、比较、拼接）
  - `#include <string>`
  - `getline(cin, s)` 读取整行
  - `s.length()`, `s[i]`, `s + s2` 等基础操作
  
- **指针**：
  - 指针基础操作
  - 指针作为函数参数
  - 数组与指针的关系
  
- **格式化输出**：
  - `#include <iomanip>`
  - `setprecision()`, `fixed`, `setw()`, `setfill()`
  
- **基础语法**：
  - 循环：for, while
  - 条件：if-else
  - 函数：递归、普通函数
  - 引用：引用作为函数参数

**❌ 不能使用（教学进度外）**：
- **STL容器**：
  - `vector`：未教过
  - `map`：未教过
  - `list`：未教过
  - 其他STL容器
  
- **复杂STL算法**：
  - `sort()`：未教过（但可以自己实现冒泡排序等）
  - `find()`：未教过（但可以自己实现查找）
  - `count()`：未教过
  - 其他STL算法

**⚠️ 注意**：
- 虽然题目中可能看到vector的使用，但那是学生自己学的，不是教学要求
- 生成题目时应该使用普通数组，不使用STL容器
- 字符串可以使用string类，但只用基础操作

#### 3. 边界处理
- 通常有边界检查
- 处理特殊情况（如除零、空输入等）

### 四、常见题型分类

#### 1. 基础语法题（简单）
- **整数操作**：求和、平方、取模
- **简单计算**：四则运算、平均值
- **条件判断**：奇偶、大小比较

**示例**：
- `整数求和.cpp`：输入两个数，输出和
- `数的平方.cpp`：输入一个数，输出平方

#### 2. 循环逻辑题（中等）
- **筛选问题**：按条件筛选数字
- **递推问题**：空瓶换水、猴子吃桃
- **统计问题**：计数、求和

**示例**：
- `筛选数字.cpp`：找第k个满足条件的数字
- `空瓶换水.cpp`：循环计算能喝多少瓶
- `素数求和.cpp`：区间内素数统计

#### 3. 数组/矩阵题（中等）
- **一维数组**：查找、统计、排序（自己实现冒泡排序等）
- **二维数组**：矩阵操作、特殊元素查找

**示例**：
- `寻找鞍点.cpp`：二维数组找鞍点（使用普通二维数组）
- `蛇形矩阵.cpp`：复杂矩阵填充（使用普通二维数组）
- `整形数组插数排序.cpp`：数组插入排序（使用普通数组）
- `二维数组求最大值.cpp`：二维数组找最大值（使用普通二维数组）

#### 4. 字符串处理题（中等）
- **字符统计**：数字、字母、空格等（使用string基础操作）
- **字符串操作**：删除、替换、反转（使用string基础操作）
- **字符串转换**：大小写、字符替换（使用`<cctype>`）

**示例**：
- `字符串统计.cpp`：统计各类字符数量（使用string和cctype）
- `删除字符.cpp`：删除指定字符（使用string基础操作）
- `句子反转.cpp`：反转句子并转换（使用string基础操作，可能用到vector但应避免）

#### 5. 指针与引用题（中等）
- **指针操作**：指针比较、指针参数
- **引用传递**：引用作为函数参数

**示例**：
- `指针比较三个数大小.cpp`：使用指针排序
- `用引用变量作为形参，实现3个整数由小到大输出.cpp`：使用引用排序
- `输入三个字符串比大小（指针与函数）.cpp`：字符串指针操作

#### 6. 进制转换题（中等）
- **十进制转N进制**：除N取余（使用string拼接）
- **N进制转十进制**：按权展开

**示例**：
- `十进制转N进制.cpp`：转换为任意进制（使用string）
- `N进制转十进制.cpp`：三位N进制转十进制

#### 7. 图形输出题（中等）
- **数字图形**：金字塔、倒金字塔
- **字符图形**：特定格式输出

**示例**：
- `N层倒金字塔.cpp`：输出倒金字塔数字图形

#### 8. OOP基础题（中等）
- **类设计**：定义简单类
- **继承**：单继承、构造函数调用
- **多态**：虚函数应用

**示例**：
- `定义一个Date类.cpp`：日期类
- `设计一个People类.cpp`：人员类（静态成员）
- `定义基类Point和派生类Circle.cpp`：继承

#### 9. 函数题（中等）
- **递归函数**：阶乘、汉诺塔、猴子吃桃
- **字符串函数**：拷贝、比较
- **数学函数**：最大公约数、回文判断

**示例**：
- `阶乘.cpp`：递归求阶乘
- `汉诺塔.cpp`：递归解决汉诺塔
- `回文数判断.cpp`：判断回文数

#### 10. 综合应用题（较难）
- **多步骤处理**：需要多轮操作
- **复杂逻辑**：状态机、模拟器

**示例**：
- `机器时代.cpp`：模拟器实现
- `师生管理系统.cpp`：OOP综合应用

### 五、难度分布

#### 简单（30%）
- 基础语法
- 单一知识点
- 代码量少（<30行）

#### 中等（50%）
- 循环+条件
- 数组/字符串操作
- 代码量中等（30-80行）

#### 较难（20%）
- 复杂逻辑
- 多步骤处理
- 代码量较多（80-150行）

### 六、函数题特殊格式

#### 1. 函数题特点
- **给出主函数**：主函数已写好
- **给出函数声明**：需要实现的函数已声明
- **填写函数体**：只需实现函数功能

#### 2. 常见格式
```cpp
// 主函数已给出
int main() {
    int n;
    while(cin>>n) {
        cout<<fact(n)<<endl;
    }
    return 0;
}

// 请在此处填写答案
long long int fact(int n) {
    // 实现函数
}
```

### 七、输出格式示例

#### 1. 简单输出
```cpp
cout << result << endl;  // 直接输出数字
```

#### 2. 格式化输出
```cpp
cout << fixed << setprecision(2) << value << endl;  // 保留2位小数
```

#### 3. 带标签输出
```cpp
cout << "数字:" << count << endl;
cout << "A等级" << countA << "人" << endl;
```

#### 4. 复杂格式输出
```cpp
cout << "The saddle point is (" << i << "," << j << ")=" << val << "." << endl;
```

### 八、关键注意事项

1. **输入输出格式必须精确**：空格、换行、标点都要准确
2. **边界条件处理**：考虑特殊情况（如除零、空输入）
3. **代码简洁**：不要过度设计，直接实现功能
4. **使用基础STL**：vector, string, algorithm足够
5. **函数题格式**：注意是填写函数体还是完整程序

## 📝 生成题目时的参考标准

1. **输入格式**：明确说明输入方式（单行/多行/文件尾）
2. **输出格式**：精确到空格、换行、标点
3. **样例**：提供清晰的样例输入输出
4. **难度适中**：符合PTA平台难度分布
5. **代码量**：控制在合理范围内（50-100行为主）

## ⚠️ 重要限制（基于教学进度）

### 必须使用
- ✅ **普通数组**：`int a[100]`, `int a[100][100]`
- ✅ **string类基础操作**：输入输出、长度、比较、拼接、字符访问
- ✅ **指针基础**：指针变量、指针参数、数组与指针
- ✅ **引用**：引用作为函数参数
- ✅ **基础算法**：自己实现排序、查找等

### 禁止使用
- ❌ **STL容器**：vector, map, list等所有STL容器
- ❌ **STL算法**：sort, find, count等STL算法
- ❌ **复杂STL**：迭代器、Lambda表达式等

### 可以使用的基础库
- ✅ `<iostream>`：输入输出
- ✅ `<string>`：string类（仅基础操作）
- ✅ `<iomanip>`：格式化输出
- ✅ `<cctype>`：字符判断函数（isdigit, isalpha等）
- ✅ `<cmath>`：数学函数（如果需要）

### 字符串操作示例（可以使用）
```cpp
string s;
cin >> s;              // ✅ 输入
getline(cin, s);       // ✅ 读取整行
s.length();            // ✅ 长度
s[i];                  // ✅ 访问字符
s + s2;                // ✅ 拼接
s == s2;               // ✅ 比较
s[i] = 'a';            // ✅ 修改字符
```

### 数组操作示例（必须使用）
```cpp
int a[100];            // ✅ 一维数组
int a[100][100];       // ✅ 二维数组
// 自己实现排序、查找等算法
```
